## 1.1 게임 개요

### 한 줄 소개
"항해사가 되어 세계를 누비며 각 나라의 문화와 특산품을 발견하는 교육 탐험 게임"

### 핵심 루프

```
항구 출발 → 방향키 항해 → 항구 근접 감지 → 발견! 연출
    ↑               ↓
 기록장 확인 ← 교육 카드 표시 → 도장 획득 → 칭호 갱신
```

1. **출발**: 초기 항구(서울/인천)에서 시작, 배를 조종
2. **항해**: 방향키/WASD로 세계 지도 위를 자유 이동
3. **근접 감지**: 항구 반경 60px 진입 시 닻 아이콘 + "A키로 발견!" 힌트 표시
4. **발견 연출**: 별 폭발 파티클 + 효과음 + 줌인 애니메이션
5. **교육 카드**: 도시명, 랜드마크, 특산품 3개, 재미있는 사실 순차 표시
6. **도장 획득**: 기록장에 방문 도장 추가, HUD 카운터 증가
7. **칭호 확인**: 발견 수 임계값 달성 시 칭호 승급 연출
8. **반복**: 지도로 돌아와 다음 항구 탐험

### 목표 플레이 시간
- **1회 세션**: 15~30분
- **전체 완성**: 5~10시간 (50개 항구 전부 발견)
- **일일 권장**: 20분 (항구 3~5개 발견)

### 학습 목표
| 목표 | 내용 |
|------|------|
| 세계 주요 도시 위치 | 50개 도시를 세계 지도 위에서 직접 찾아가며 위치 학습 |
| 각 나라 특산품 | 도시당 특산품 3개, 실물 이미지 카드로 연상 학습 |
| 문화 사실 | 랜드마크, 역사적 사실, 음식 등 흥미로운 정보 |
| 이중 언어 | 모든 도시명·특산품명 한국어/영어 병기 |

## 1.2 캐릭터 시스템

### 캐릭터 A — 항해사 준 (Jun the Navigator)

| 항목 | 내용 |
|------|------|
| 이름 | 준 / Jun |
| 직함 | 항해사 (Navigator) |
| 외형 | 파란 선장 모자, 파란 줄무늬 셔츠, 밝은 갈색 피부, 검은 단발 |
| 성격 | 밝고 호기심 많음, "우와! 여기가 어디야?" 타입 |
| 말투 | 신나고 에너지 넘치는 구어체 |
| 배 색상 | 파란색 범선 |
| 특징 | 나침반 아이콘이 항상 손에 들려 있음 |

**준의 대사 예시**
- 게임 시작: "안녕! 나는 준이야. 같이 세계를 탐험하자!"
- 항구 발견: "드디어 찾았다! 여기가 바로 [도시명]이구나!"
- 칭호 승급: "와아! 이제 나는 진짜 탐험가야!"

### 캐릭터 B — 탐험가 아라 (Ara the Explorer)

| 항목 | 내용 |
|------|------|
| 이름 | 아라 / Ara |
| 직함 | 탐험가 (Explorer) |
| 외형 | 빨간 두건, 초록 탐험복, 어두운 갈색 피부, 긴 흑발 묶음 |
| 성격 | 용감하고 지식욕 강함, "이 나라엔 어떤 비밀이 있을까?" 타입 |
| 말투 | 차분하지만 열정적, 약간 학구적 |
| 배 색상 | 빨간색 범선 |
| 특징 | 항상 탐험 노트를 들고 다님 |

**아라의 대사 예시**
- 게임 시작: "안녕, 나는 아라야. 세계의 비밀을 함께 풀어볼까?"
- 항구 발견: "여기가 [도시명]이구나. 노트에 기록해야겠어!"
- 칭호 승급: "더 넓은 세계가 기다리고 있어. 계속 탐험하자!"

### 캐릭터 선택 화면 상세 설계

**레이아웃**: 두 캐릭터 카드를 좌우로 배치, 가운데 VS 마크 없이 "누구와 함께 떠날까?" 문구

**카드 구성 요소** (각 카드):
- 캐릭터 전신 일러스트 (256x384px)
- 이름 (한/영, 큰 폰트)
- 직함 배지
- 성격 설명 3줄 이내
- 배 미니 일러스트
- "선택하기" 버튼

**인터랙션**:
- 카드 hover: 살짝 위로 떠오르는 효과 (y -8px, tweened)
- 카드 클릭: 선택된 카드 테두리 강조 → 0.5초 후 "출항!" 확인 다이얼로그
- 확인 버튼 누르면 WorldMapScene으로 전환 (페이드 아웃)

**하단 추가 요소**:
- "어떤 탐험가가 될지 골라봐!" 안내 문구 (작은 폰트)
- 두 캐릭터 모두 똑같이 50개 항구를 탐험 가능 — 차이는 외형과 대사뿐

## 1.3 세계지도 화면 (WorldMapScene)

### 세계지도 시각화

**캔버스 크기**: 1024 x 512px (Phaser 3 게임 해상도)  
**지도 이미지**: 1024 x 512 메르카토르 투영 세계 지도 PNG  
- 육지: 연한 초록/베이지 (#d4e8b0 계열)
- 바다: 연한 파랑 (#a8d4f0 계열)
- 스크롤 없음 — 전체 세계 지도가 화면에 딱 맞게 고정

**좌표계**: 위경도 → 픽셀 변환 공식
```typescript
x = (longitude + 180) / 360 * 1024
y = (90 - latitude) / 180 * 512
```

### 배 이동

**조작**: 방향키(↑↓←→) 또는 WASD  
**속도**: 기본 120px/s  
**배 스프라이트**: 32x32px, 이동 방향에 따라 회전 (8방향)  
**경계 처리**: 좌우 랩어라운드 (서쪽 끝 → 동쪽 끝), 상하 반전 없이 경계에서 정지  
**대각선 이동**: 지원 (두 방향키 동시 입력), 속도 동일 유지 (normalize)  
**물 위만 이동**: 육지 충돌 없음 — 초등학생 대상이므로 장애물 없이 자유 이동

### 항구 표시 상태

| 상태 | 시각 표현 |
|------|----------|
| 미발견 기본 | 노란 점 (8px 원, #FFD700) — 지도에 희미하게 보임 |
| 근접 (60px 이내) | 닻 아이콘 (24px) + 흰색 글로우 펄스 애니메이션 + "A키로 발견!" 툴팁 |
| 발견 완료 | 작은 깃발 아이콘 (16px) + 대륙별 색상으로 변경 |

**대륙별 깃발 색상**:
- 유럽: 파랑 (#4477FF)
- 아시아: 빨강 (#FF4444)
- 아메리카: 초록 (#44BB44)
- 아프리카: 주황 (#FF8822)
- 오세아니아: 보라 (#AA44FF)

### 미니맵
없음 — 화면 자체가 세계 전체 지도이므로 미니맵 불필요

### HUD 구성
화면 상단 바 (전체 가로 × 40px 높이):

```
[배 아이콘] 준의 대항해  |  🚀 발견: 12/50  |  🏆 탐험가  |  [기록장 버튼]
```

- 좌측: 캐릭터 이름 + 배 아이콘
- 중앙: 발견 카운터 (큰 숫자, 강조)
- 중앙우: 현재 칭호 배지
- 우측: 기록장 열기 버튼 (📖 아이콘 + "기록장" 텍스트)

### 발견 연출 순서
1. 배가 항구 반경 진입 → 닻 아이콘 + 힌트 표시
2. A키 또는 스페이스 입력
3. 화면 전체 잠깐 화이트 플래시 (0.1s)
4. 별 파티클 폭발 (항구 위치 기준)
5. HUD 카운터 증가 (숫자 튐 애니메이션)
6. PortScene으로 전환 (슬라이드 업)

## 1.4 항구 카드 UI (PortScene)

### 카드 레이아웃 (ASCII 와이어프레임)

```
+========================================+
|  ✕ 닫기                               |
|                                        |
|  +----------------------------------+  |
|  |   [랜드마크 일러스트 160x120px]  |  |
|  +----------------------------------+  |
|                                        |
|   도쿄 / Tokyo          🇯🇵           |
|   일본 수도 · 아시아                   |
|                                        |
|  ┌──────────────────────────────────┐  |
|  │  특산품                          │  |
|  │  🍣 초밥 / Sushi                 │  |
|  │  🌸 벚꽃 / Cherry Blossom       │  |
|  │  📺 전자제품 / Electronics      │  |
|  └──────────────────────────────────┘  |
|                                        |
|  💡 알고 있었나요?                     |
|  도쿄는 세계에서 인구가 가장 많은     |
|  도시 중 하나예요! 약 1400만 명이      |
|  살고 있답니다.                        |
|                                        |
|         [  다음 탐험 계속하기 →  ]     |
+========================================+
```

### 카드 표시 순서 (단계별 등장)

1. **도시명** (한국어 대형 + 영어 소형): 0.3s 페이드인
2. **랜드마크 이미지**: 0.4s 줌인 (scale 0.8 → 1.0)
3. **특산품 3개**: 0.2s 간격으로 순차 슬라이드인 (좌→우)
4. **재미있는 사실**: 0.5s 지연 후 타이핑 효과 (글자 하나씩 표시)
5. **"다음" 버튼**: 모든 내용 표시 완료 후 0.5s 지연 후 나타남

### 카드 애니메이션

**등장**: PortScene 전환 시 카드 전체가 화면 하단에서 슬라이드 업 (y: 600 → 100, ease: Back.easeOut, duration: 400ms)

**닫기(다음 탐험)**: 
- 버튼 클릭 시 카드 슬라이드 다운 (y: 100 → 600, ease: Back.easeIn, 300ms)
- WorldMapScene으로 복귀

**✕ 닫기 버튼**: 카드 우상단 — 교육 내용 읽지 않아도 건너뛸 수 있음 (강제 학습 금지)

### 카드 색상 테마 (대륙별)
- 아시아: 따뜻한 빨강/금색 그라데이션
- 유럽: 파랑/은색
- 아메리카: 초록/갈색
- 아프리카: 주황/황토색
- 오세아니아: 청록/하늘색

### "다음" 버튼 위치
카드 최하단 중앙, 전체 카드 너비의 60% 크기, 두드러지는 색상 (#FF8C00 주황)

### 방문 도시 그리드
- 카드 크기: 100x130px
- 카드 내용: 랜드마크 썸네일 이미지(70x70px) + 도시명(한국어) + 도시명(영어, 작은 폰트)
- 발견한 도시: 풀 컬러 + 대륙별 테두리 색상
- 미발견 도시: 회색 실루엣 + "?" 마크 (도시 개수는 보여주지만 어디인지 모름)
- 그리드 레이아웃: 5열, 스크롤 가능 (Phaser 스크롤 마스크 사용)
- 카드 클릭 시: 이미 방문한 도시면 간단 팝업으로 정보 재표시

### 세계 지도 탭
- 800x400 축소 세계 지도
- 방문한 도시 위치에 깃발 아이콘 표시
- 방문한 대륙/지역 위에 반투명 색상 오버레이 (alpha: 0.3)
- 미방문 지역은 흑백 처리

### 진행률 표시
- 전체 진행률 바: 카드 상단 (12/50, 진행률 24%)
- 대륙별 미니 진행 바: 세계 지도 탭 좌측 사이드바
- 칭호 다음 단계까지 남은 수 표시: "탐험가 → 항해사까지 앞으로 4개!"

## 1.6 칭호 시스템

### 칭호 단계표

| 단계 | 발견 수 | 칭호 (한국어) | 칭호 (영어) | 아이콘 | 색상 |
|------|---------|--------------|------------|-------|------|
| 1 | 0~5 | 견습 선원 | Apprentice Sailor | 🚢 | 회색 |
| 2 | 6~15 | 탐험가 | Explorer | ⚓ | 초록 |
| 3 | 16~30 | 항해사 | Navigator | 🗺️ | 파랑 |
| 4 | 31~50 | 세계 일주 달인 | World Voyager | 🌍 | 금색 |

### 칭호 변경 이펙트 (전용 연출 씬)

임계값 도달 시 WorldMapScene 위에 오버레이로 연출:

1. 화면 전체에 반투명 다크 오버레이 (alpha 0 → 0.7, 0.5s)
2. 새 칭호 배지가 화면 중앙에서 팝업 (scale 0 → 1.2 → 1.0, Back.easeOut)
3. 금색 빛 파티클 100개 폭발
4. 칭호명 큰 글씨로 등장 (트럼펫 효과음)
5. 서브 텍스트: "이제 [새 칭호]이 되었어요!"
6. 3초 후 자동 닫힘 또는 탭/클릭으로 즉시 닫힘

**HUD 칭호 배지 갱신**: 칭호 변경 후 HUD의 칭호 배지도 새 칭호로 부드럽게 교체

## 1.7 저장 시스템 (localStorage)

### 저장 데이터 구조

```typescript
interface SaveData {
  character: 'jun' | 'ara';           // 선택한 캐릭터
  visitedPorts: string[];             // 방문한 항구 ID 배열 (예: ['tokyo', 'paris'])
  shipPosition: { x: number; y: number }; // 현재 배 위치 (픽셀 좌표)
  totalDiscovered: number;            // 발견 총 개수 (캐시)
  lastPlayed: string;                 // ISO 날짜 문자열
}
```

**저장 키**: `worldExplorer_save`

### 저장 타이밍
- **새 항구 발견 시**: PortScene에서 "다음 탐험 계속하기" 버튼 누를 때 저장
- **저장 시 표시**: HUD에 작은 "💾 저장됨" 텍스트 0.5초간 표시

### 불러오기
- TitleScene에서 localStorage 키 존재 확인
- 저장 데이터 있으면 "이어하기" 버튼 활성화
- 이어하기 선택 시 CharacterSelectScene 생략하고 WorldMapScene으로 직행

### 초기화
- TitleScene 하단에 작은 "처음부터 다시" 버튼 (실수 방지용 확인 다이얼로그 포함)

## 1.8 UI 전체 흐름 (씬 전환 다이어그램)

```
[TitleScene]
    │
    ├── 새 게임 ──────────→ [CharacterSelectScene]
    │                               │
    └── 이어하기 ──────────────────→ [WorldMapScene] ←─────────────┐
                                        │                          │
                              항구 근접 + A키                      │
                                        │                          │
                                        ↓                          │
                                  [PortScene]                      │
                                  (교육 카드)                      │
                                        │                          │
                              "다음 탐험" 버튼          "뒤로" 버튼
                                        │                          │
                                        └────────────────────────→─┘
                                                                   │
                         [기록장 버튼] ──→ [LogbookScene] ─────────┘
```

| LogbookScene → WorldMap | 페이드 아웃/인 (0.3s) |
